

## Fix: Show OTP Code in Email Instead of Magic Link

### Problem
The current email sent by `signInWithOtp` uses the default template which shows a "Log In" button (magic link) instead of the 6-digit OTP code. The code is generated by the system but the email template doesn't display it.

### Solution
Scaffold custom auth email templates and customize the **magic-link** template to prominently display the 6-digit OTP code instead of the login link.

### Steps

1. **Scaffold auth email templates** using the email domain tool (domain `inhalestays.com` is already configured, DNS verification not required for scaffolding)

2. **Customize the magic-link template** (`supabase/functions/_shared/email-templates/magic-link.tsx`):
   - Remove the "Log In" button/link
   - Display the 6-digit OTP code prominently in large, styled text
   - Update copy: "Your verification code is: **123456**" instead of "Click the button below..."
   - Apply brand colors from the project's CSS variables

3. **Deploy the `auth-email-hook` edge function**

4. **No code changes needed in `StudentLogin.tsx`** — the OTP verification logic already works correctly, only the email content needs to change

### Files Created/Modified
- `supabase/functions/auth-email-hook/index.ts` (scaffolded)
- `supabase/functions/_shared/email-templates/magic-link.tsx` (customized to show OTP code)
- All other scaffolded templates get brand styling applied

### Note
DNS for `inhalestays.com` is still pending verification. Until verified, the default Lovable emails will continue to be sent. Once DNS is verified, the custom templates with the OTP code will activate automatically.
